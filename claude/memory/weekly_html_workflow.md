### HTML Structure

Each LOB HTML has **two section blocks**:
1. **In-House** (LOB-colored badge: consumer=#1565c0, dasher=#2e7d32, integrity=#6a1b9a)
   - HC only (no FTE billing)
   - Source: `in_house/2026-04-20.csv` filtered by section (Cx/Dx/Integrity)
   - AOP from `in_house/aop.csv`
   - NEW clusters shown in purple italic (AOP blank = "—" everywhere)
2. **BPO** (red badge #FF3008)
   - Both FTE Billing and HC Total
   - Parses txt files: `{lob}_fte_billing_cq.txt`, `{lob}_fte_billing_h2.txt`, etc.

### Key Design Decisions
- **Why:** Audience is Gale (Dasher finance) and Finance team — want both in-house and BPO in one view
- **H2 format**: 4 sub-rows per cluster (Lock / AOP / vs AOP / vs AOP%)
- **Flag threshold**: ≥5% vs AOP for both in-house and BPO
- **Split H2 line fix**: `re.sub(r'◄\s*', '◄  ', rest)` normalizes spacing before splitting, fixes jul-column-jamming bug

### Data Sources (as of Apr 20 lock)
- In-house lock: `in_house/2026-04-20.csv` (rename to current date each week)
- In-house AOP: `in_house/aop.csv` (stable, update if AOP changes)
- BPO txt files: generated by `build_*_tables.py` scripts each week
